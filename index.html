1. MD 내용 불변 원칙
   MD 파일의 텍스트, 구조, 데이터를 임의로 변경·추가·삭제하지 마.
   오직 있는 그대로 HTML로 변환해야 해. 어떤 이유로도 내용을 수정하면 안 돼.

2. UI/UX 개선 제안 제외
   MD 파일에 UI/UX 개선 제안 섹션이 포함되어 있더라도 HTML 변환 시 넣지 마.

3. 기능구분 ID 앵커 링크
   기능 구분 목록 테이블의 기능구분 ID(예: BIZ-001, WKP-001)를
   클릭 시 해당 그룹 섹션으로 이동하는 앵커 링크로 구성해.

   기존 HTML의 사이드바, 카드, 테이블, 컬러 시스템을 그대로 유지해.

5. 단일 메뉴 구성
   MD 파일 전체를 하나의 메뉴 항목으로 사이드바에 추가해.
   내부 세부 기능을 별도 사이드바 항목으로 나누지 마.

6. 업데이트 내역 갱신
   메뉴 01(업데이트 내역)에 오늘 날짜와 추가 내용을 기재해.
   최신순 5개 항목만 유지해. 6번째부터는 제거해.

7. 플로팅 버튼 유지 (필수)
   HTML 파일 우측 상단에 고정된 "컴포넌트 정의서 바로가기" 플로팅 버튼을
   절대 제거하거나 수정하지 마. 아래 코드를 사이드바 닫힘 태그와 메인 태그 사이에 그대로 유지해.
